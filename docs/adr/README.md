# Architecture Decision Records

This directory will store Architecture Decision Records (ADRs) as they are created. 

ADRs track significant architectural decisions, their context, consequences, and alternative options.

Example records to be added in future sprints:
- `ADR-001-why-mysql.md`
- `ADR-002-why-modular-monolith.md`
- `ADR-003-why-spotless.md`
- `ADR-004-why-standard-api-response.md`
